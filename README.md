基于 [AstrBot](https://github.com/Soulter/AstrBot) 的群聊互动插件。通过记录群内活跃成员（30天内发言的用户），实现随机抽取「今日老婆」的功能，并配备好感度系统、恋爱系统、日群友系统与关系图谱可视化。

## ✨ 功能亮点

* **活跃筛选**：仅从 30 天内有发言记录的「活人」中抽取，自动过滤机器人与 ID 为 0 的异常账号。
* **头像展示**：抽取结果附带 640px 高清 QQ 头像，视觉体验更佳。
* **好感度系统**：每次强娶增加好感度，达到 100% 可触发恋爱绑定，每月自动重置并有自然衰减。
* **恋爱系统**：恋爱绑定后双方今日老婆固定为对方，无法被强娶，支持邀请与拒绝流程。
* **CP 双向保护**：每对 CP 首次互动锁定一个 50~100% 的随机阈值，好感度跨过即在 animewifex 那边也保护双方（互不能被牛/被换）。
* **CG 合成**：支持将两位群友合成至自定义 CG 图，需双方有好感度记录。
* **日群友**：独立的日群友互动系统，支持随机/指定模式及月榜排行。
* **每日玩法提示**：抽老婆、强娶、日群友等结果会追加下一步互动提示，减少“一抽就走”。
* **使用数据看板**：管理员可查看近 7 天使用人数、热门命令和回访概况，方便判断群内玩法热度。
* **animewifex 强联动**：发「抽老婆」一条命令同时出群友老婆 + 二次元老婆，命中本命同担/命中注定时强娶有 buff，资源与对面互喂。
* **可视化关系图叠层**：关系图每个节点会挂当日二次元老婆缩略图作为卫星节点，本命同担群友间画金色虚线连接。
* **智能名称识别**：图谱自动关联用户昵称，优先显示昵称而非数字 ID。
* **灵活管控**：支持群聊白名单与黑名单，以及每人每日抽取次数限制。

## 🔥 留存玩法闭环

这个版本默认把插件调成更适合群聊日常滚动的节奏：每日老婆次数从 1 次提升到 3 次，关键词触发默认开启，强娶默认改为每日次数制，结果消息会继续引导用户去看状态、强娶、日群友或查好感度。

推荐群内玩法顺序：

1. 发 `抽老婆` 开始今日老婆。
2. 发 `强娶 @群友` 抢人并推进好感度。
3. 发 `日群友` 或 `日群友 @群友` 制造更多群内互动。
4. 发 `今日玩法` 看本群今日活跃池、老婆、强娶、日群友和好感度概况。
5. 管理员发 `老婆插件数据` 看近 7 天使用人数、热门命令和回访情况。

如果想让插件更“吵一点”，可以保持 `keyword_trigger_enabled=true` 和 `keyword_trigger_mode=starts_with`。如果群里比较严肃，建议改成 `exact`，只响应完整关键词。

## 🔗 animewifex 强联动

两个插件都装在同一个 AstrBot 实例就自动生效，无需手动开关。通过模块级 `get_instance()` 单例互相 try-import，对方插件不在时静默降级。

### 🎴 一次抽老婆，两张老婆

发 **`抽老婆`** 时，animewifex 先抽今日二次元老婆 → wifepicker 在后台静默抽一位今日群友 → 两条消息合并出。
- 二次元老婆图 + 立绘
- 群友老婆 + QQ 头像
- 命中同担时附 buff 提示

### ✨ 本命同担 buff

如果抽到的群友（或群里任何群友）和你的**本命列表**有重叠，或者你/TA 的本命就是对方今日的二次元老婆 → 触发同担：
- 抽老婆消息会列出**前 5 位同担群友** + 各自随机倍率
- 对这些群友任意一位发 `强娶` → 好感度增益 **×1.3 ~ ×2.0** 随机加成（倍率每对独立锁定）
- 全名单可发 `查同担` 查看；`查同担 @某人` 单独对照本命

### 🌟 命中注定

强娶时如果你的本命角色 = 对方当天的二次元老婆（或反过来）→ 触发"命中注定"：
- 好感度直接 **+30**（盖过同担 buff）
- 双方在 animewifex 各获得当日称号「命中注定·YYYYMMDD」

### 🔒 业力镜像保护

animewifex 业力惩罚锁的人，wifepicker 这边当天**免被强娶**——一边受惩罚两边都不挨打。

### 💞 CP 双向保护

每对 CP 首次互动时锁定一个 **50~100%** 的随机好感度阈值，跨过阈值即生效：
- wifepicker 走原有恋爱绑定保护
- animewifex 那边对方**不能被你 `牛老婆` / `交换老婆` 抢走**
- 100% 好感度（恋爱绑定）的两人在二次元也互不能抢

### 💔 NTR 传染

animewifex 里某人被 `牛老婆` 成功 → wifepicker 这边被牛者**强娶冷却 +30 分钟**（daily 模式扣 1 次次数）。家里媳妇被牛了，没心情强娶人。

### 🎁 资源互喂

- wifepicker 好感度首次满 100% → 双方各送 animewifex 补签券 ×1
- animewifex 连签里程碑（7/30/100 天）→ wifepicker 强娶次数 +1

### 🎴 关系图叠层

`关系图` 现在叠两层：
- 每位群友节点旁挂一个**卫星节点**，圆形头像 = 当日二次元老婆立绘（紫色边框）
- 本命同担群友间画一条**金色虚线**，标 `本命同担 ×x.x`

## 🎮 使用指令

> ⚠️ 使用强娶并 @ 对方时，请**长按对方头像 @**，或点击 QQ 弹出的名字列表中的 @，**不要直接输入名字或复制粘贴**。只有正确 @ 才能让插件获取到 QQ 号，这是所有 AstrBot 插件的通用限制。

### 抽老婆

| 指令 | 别名 | 权限 | 说明 |
| --- | --- | --- | --- |
| `/今日老婆` | `抽老婆` | 用户 | 随机抽取今日老婆（恋爱中则直接显示伴侣）。**装了 animewifex 时，由 animewifex 端触发，会同时出二次元老婆 + 群友老婆 + 同担列表** |
| `/强娶 @用户` | — | 用户 | 消耗次数强制与指定用户建立羁绊，并增加好感度（同担/命中注定 buff 自动生效） |
| `/我的老婆` | `抽取历史` | 用户 | 查看今日抽取记录与剩余次数 |
| `/今日玩法` | `老婆状态` / `老婆日报` | 用户 | 查看本群今日活跃池、抽取、强娶、日群友、本命同担和好感度概况 |
| `/查同担` | `本命同担` / `同担列表` | 用户 | 列出本群所有跟你本命有重叠的人；带 @ 时只对照指定用户 |
| `/关系图` | — | 用户 | 生成并发送本群今日老婆关系网络图（自动叠加二次元老婆 + 同担连线） |
| `/rbq排行` | — | 用户 | 展示近 30 天被强娶次数排行（前 10 名） |
| `/抽老婆帮助` | `老婆插件帮助` | 用户 | 查看插件内置的指令说明 |
| `/老婆插件数据` | `老婆数据` / `老婆留存` | 管理员 | 查看本群近 7 天使用人数、热门命令和回访概况 |
| `/重置记录` | — | 管理员 | 清空本群今日所有抽取记录 |
| `/重置强娶时间` | — | 管理员 | 重置本群所有强娶冷却（含上锁记录） |
| `/重置强娶次数` | — | 管理员 | 重置本群今日强娶次数（冷却时间不变，仅 daily 模式） |

### 好感度 & 恋爱

| 指令 | 权限 | 说明 |
| --- | --- | --- |
| `/好感度` | 用户 | 查看自己的今日老婆及所有 CP 的本月好感度 |
| `/好感度 @用户` | 用户 | 查看与指定用户的好感度进度条 |
| `/好感度排行` | 用户 | 展示本群本月 CP 好感度排行（前 10） |
| `/恩爱排行` | 用户 | 展示本月最先达成 100% 好感度的 CP 排行 |
| `/CG @A @B 文件名.png` | 用户 | 将两位群友合成到指定 CG 图（需双方有好感度记录）|
| `/好感度调整 @A @B +N` | 管理员 | 调整指定 CP 的好感度，支持 `+N` / `-N` / `=N`，范围 0~100 |

> CG 指令支持指定左右：`/CG @A左 @B右 文件名.png`，默认 A 左 B 右。

### 日群友

| 指令 | 权限 | 说明 |
| --- | --- | --- |
| `/日群友` | 用户 | 随机日一位群友（每人每日限额） |
| `/日群友 @用户` | 用户 | 指定日某位群友 |
| `/日群友排行` | 用户 | 近 30 天日群友次数排行（前 10） |
| `/日群友关系图` | 用户 | 生成本群今日日群友关系图 |

> 默认开启 `keyword_trigger_enabled`，常用用户指令可作为关键词触发，无需 `/` 前缀。
> 管理员指令需要使用带前缀的正式指令触发，避免关键词绕过权限校验。
> 若在 AstrBot 指令管理中禁用某条指令，对应关键词触发也会同步失效。

## 🛠️ 安装环境要求

本插件的关系图功能依赖 AstrBot 的浏览器渲染引擎（一般已内置，无需额外安装）：

1. 确保 AstrBot 环境已正确安装 `Playwright` 浏览器驱动。
2. 插件目录下需包含 `graph_template.html`、`ri_graph_template.html`、`rbq_ranking.html`、`ri_ranking.html`、`affinity_ranking.html` 等模板文件。
3. 渲染使用本地内嵌的 `vis-network.min.js`，无需联网加载。
4. CG 功能需在插件目录下创建 `cg/` 子目录并放入对应 PNG 模板文件。

## ⚙️ 配置项说明

在 AstrBot 管理面板或 `_conf_schema.json` 中可配置以下内容：

### 基础配置

| 配置键 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `daily_limit` | int | 3 | 每人每天可抽取今日老婆的次数上限 |
| `max_records` | int | 2000 | 全局 JSON 存储的最大记录条数 |
| `excluded_users` | list | [] | 永远不会被抽中的 QQ 号列表（今日老婆池排除） |
| `force_marry_excluded_users` | list | [] | 强娶排除列表，列表中的用户无法被强娶 |
| `whitelist_groups` | list | [] | 白名单模式：仅在此列表中的群生效 |
| `blacklist_groups` | list | [] | 黑名单模式：列表中的群禁用插件 |
| `auto_set_other_half` | bool | true | 自动设置对方老婆（对方当天无记录时才会生效） |
| `auto_withdraw_enabled` | bool | false | 定时自动撤回消息（仅 aiocqhttp/OneBot 可用） |
| `auto_withdraw_delay_seconds` | int | 5 | 自动撤回延迟秒数 |
| `engagement_hints_enabled` | bool | true | 在结果中追加下一步玩法提示 |

> animewifex 联动不需要配置开关——两个插件同时存在就自动生效。旧版 `animewifex_link_*` / `animewifex_config_dir` 字段已弃用。

### 关键词触发配置

| 配置键 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `keyword_trigger_enabled` | bool | true | 是否启用关键词触发（无需 `/` 前缀） |
| `keyword_trigger_mode` | string | `starts_with` | 关键词匹配模式：`exact` / `starts_with` / `contains` |

### 日群友配置

| 配置键 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `ri_probability` | float | 80 | 随机日群友成功概率，失败不消耗额度 |
| `ri_at_probability` | float | 80 | @指定日群友成功概率，失败不消耗额度 |
| `ri_target_max` | int | 3 | 每人每天最多被 @ 指定日次数 |
| `ri_invite_max` | int | 3 | 每人每天最多跟日次数 |

### 强娶配置

| 配置键 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `force_marry_cd_mode` | string | `daily` | 强娶冷却模式：`days`（天数CD）/ `daily`（每日次数） |
| `force_marry_cd` | int | 1 | days 模式下的冷却天数 |
| `force_marry_daily_limit` | int | 2 | daily 模式下每人每日强娶次数上限 |
| `force_marry_wife_limit` | int | 2 | 每人每日可拥有的强娶老婆上限 |
| `force_marry_lock_count` | int | 2 | 单日被强娶达此次数后触发上锁保护 |

### 好感度 & 恋爱配置

| 配置键 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `affinity_bypass_lock_threshold` | int | 50 | 好感度达此值可无视上锁保护直接强娶 |
| `force_marry_unlock_ri_count` | int | 3 | 被日群友指定日满此次数后解除上锁 |
