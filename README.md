基于 [AstrBot](https://github.com/Soulter/AstrBot) 的群聊互动插件。通过记录群内活跃成员（30天内发言的用户），实现随机抽取「今日老婆」的功能，并配备好感度系统、恋爱系统、日群友系统与关系图谱可视化。

## ✨ 功能亮点

* **活跃筛选**：仅从 30 天内有发言记录的「活人」中抽取，自动过滤机器人与 ID 为 0 的异常账号。
* **头像展示**：抽取结果附带 640px 高清 QQ 头像，视觉体验更佳。
* **好感度系统**：每次强娶增加好感度，达到 100% 可触发恋爱绑定，每月自动重置并有自然衰减。
* **恋爱系统**：恋爱绑定后双方今日老婆固定为对方，无法被强娶，支持邀请与拒绝流程。
* **CG 合成**：支持将两位群友合成至自定义 CG 图，需双方有好感度记录。
* **日群友**：独立的日群友互动系统，支持随机/指定模式及月榜排行。
* **可视化关系**：基于 `Vis.js` 渲染生成高清关系网络图，直观展示群内的老婆关系与日群友关系。
* **智能名称识别**：图谱自动关联用户昵称，优先显示昵称而非数字 ID。
* **灵活管控**：支持群聊白名单与黑名单，以及每人每日抽取次数限制。

## 🎮 使用指令

> ⚠️ 使用强娶并 @ 对方时，请**长按对方头像 @**，或点击 QQ 弹出的名字列表中的 @，**不要直接输入名字或复制粘贴**。只有正确 @ 才能让插件获取到 QQ 号，这是所有 AstrBot 插件的通用限制。

### 抽老婆

| 指令 | 别名 | 权限 | 说明 |
| --- | --- | --- | --- |
| `/今日老婆` | `抽老婆` | 用户 | 随机抽取今日老婆（恋爱中则直接显示伴侣） |
| `/强娶 @用户` | — | 用户 | 消耗次数强制与指定用户建立羁绊，并增加好感度 |
| `/我的老婆` | `抽取历史` | 用户 | 查看今日抽取记录与剩余次数 |
| `/关系图` | — | 用户 | 生成并发送本群今日老婆关系网络图 |
| `/rbq排行` | — | 用户 | 展示近 30 天被强娶次数排行（前 10 名） |
| `/抽老婆帮助` | `老婆插件帮助` | 用户 | 查看插件内置的指令说明 |
| `/重置记录` | — | 管理员 | 清空本群今日所有抽取记录 |
| `/重置强娶时间` | — | 管理员 | 重置本群所有强娶冷却（含上锁记录） |
| `/重置强娶次数` | — | 管理员 | 重置本群今日强娶次数（冷却时间不变，仅 daily 模式） |

### 好感度 & 恋爱

| 指令 | 权限 | 说明 |
| --- | --- | --- |
| `/好感度` | 用户 | 查看自己的今日老婆及所有 CP 的本月好感度 |
| `/好感度 @用户` | 用户 | 查看与指定用户的好感度进度条 |
| `/好感度排行` | 用户 | 展示本群本月 CP 好感度排行（前 10） |
| `/恩爱排行` | 用户 | 展示本月最先达成 100% 好感度的 CP 排行 |
| `/CG @A @B 文件名.png` | 用户 | 将两位群友合成到指定 CG 图（需双方有好感度记录）|
| `/好感度调整 @A @B +N` | 管理员 | 调整指定 CP 的好感度，支持 `+N` / `-N` / `=N`，范围 0~100 |

> CG 指令支持指定左右：`/CG @A左 @B右 文件名.png`，默认 A 左 B 右。

### 日群友

| 指令 | 权限 | 说明 |
| --- | --- | --- |
| `/日群友` | 用户 | 随机日一位群友（每人每日限额） |
| `/日群友 @用户` | 用户 | 指定日某位群友 |
| `/日群友排行` | 用户 | 近 30 天日群友次数排行（前 10） |
| `/日群友关系图` | 用户 | 生成本群今日日群友关系图 |

> 若在插件配置中开启 `keyword_trigger_enabled`，以上指令均可作为关键词触发，无需 `/` 前缀。
> 关键词触发遵循相同权限控制，管理员指令仍仅管理员可用。
> 若在 AstrBot 指令管理中禁用某条指令，对应关键词触发也会同步失效。

## 🛠️ 安装环境要求

本插件的关系图功能依赖 AstrBot 的浏览器渲染引擎（一般已内置，无需额外安装）：

1. 确保 AstrBot 环境已正确安装 `Playwright` 浏览器驱动。
2. 插件目录下需包含 `graph_template.html`、`ri_graph_template.html`、`rbq_ranking.html`、`ri_ranking.html`、`affinity_ranking.html` 等模板文件。
3. 渲染使用本地内嵌的 `vis-network.min.js`，无需联网加载。
4. CG 功能需在插件目录下创建 `cg/` 子目录并放入对应 PNG 模板文件。

## ⚙️ 配置项说明

在 AstrBot 管理面板或 `_conf_schema.json` 中可配置以下内容：

### 基础配置

| 配置键 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `daily_limit` | int | 1 | 每人每天可抽取今日老婆的次数上限 |
| `max_records` | int | 500 | 全局 JSON 存储的最大记录条数 |
| `excluded_users` | list | [] | 永远不会被抽中的 QQ 号列表（今日老婆池排除） |
| `force_marry_excluded_users` | list | [] | 强娶排除列表，列表中的用户无法被强娶 |
| `whitelist_groups` | list | [] | 白名单模式：仅在此列表中的群生效 |
| `blacklist_groups` | list | [] | 黑名单模式：列表中的群禁用插件 |
| `auto_set_other_half` | bool | false | 自动设置对方老婆（对方当天无记录时才会生效） |
| `auto_withdraw_enabled` | bool | false | 定时自动撤回消息（仅 aiocqhttp/OneBot 可用） |
| `auto_withdraw_delay_seconds` | int | 5 | 自动撤回延迟秒数 |

### 关键词触发配置

| 配置键 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `keyword_trigger_enabled` | bool | false | 是否启用关键词触发（无需 `/` 前缀） |
| `keyword_trigger_mode` | string | `contains` | 关键词匹配模式：`exact` / `starts_with` / `contains` |

### 强娶配置

| 配置键 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `force_marry_cd_mode` | string | `days` | 强娶冷却模式：`days`（天数CD）/ `daily`（每日次数） |
| `force_marry_cd` | int | 3 | days 模式下的冷却天数 |
| `force_marry_daily_limit` | int | 2 | daily 模式下每人每日强娶次数上限 |
| `force_marry_wife_limit` | int | 1 | 每人每日可拥有的强娶老婆上限 |
| `force_marry_lock_count` | int | 2 | 单日被强娶达此次数后触发上锁保护 |

### 好感度 & 恋爱配置

| 配置键 | 类型 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `affinity_bypass_lock_threshold` | int | 50 | 好感度达此值可无视上锁保护直接强娶 |
| `force_marry_unlock_ri_count` | int | 3 | 被日群友指定日满此次数后解除上锁 |
